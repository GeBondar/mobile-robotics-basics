# **Туториал по ROS 2: Добавляем физические и коллизионные свойства в URDF**
*Учимся делать модель робота пригодной для симуляции*

## Цель туториала
Научиться добавлять в URDF-описание робота три важных элемента:
1.  **`<collision>`** — геометрию для обнаружения столкновений.
2.  **`<inertial>`** — инерционные характеристики (масса, тензор инерции).
3.  **`<dynamics>`** — параметры движения суставов (трение, демпфирование).

---

## Зачем нужны физические свойства?
До сих пор наши модели были чисто **визуальными**. Они отлично смотрелись в Rviz, но для симуляции в физическом движке (например, Gazebo) этого недостаточно. Чтобы робот мог падать, сталкиваться с препятствиями и реагировать на силы, нужно определить:

*   **Коллизионную геометрию** — упрощённую форму деталей для быстрого расчёта столкновений. Использование сложных 3D-мешей для коллизий очень ресурсозатратно.
*   **Инерционные характеристики** — массу и распределение массы (тензор инерции). Они определяют, как робот будет реагировать на приложенные силы.
*   **Динамику суставов** — трение и демпфирование в суставах, влияющие на реалистичность движений.

---

## Предварительные требования
1.  Понимание структуры URDF (линки, джойнты).
2.  Выполнены предыдущие туториалы по созданию визуальной и подвижной моделей.
3.  Базовые знания физики (масса, инерция, трение).

## Практическая часть: физические свойства R2D2

Мы будем работать с моделью R2D2 из предыдущих туториалов. Рассмотрим, как изменилось описание его основания (`base_link`).

### **1. Добавление коллизионной геометрии (`<collision>`)**

Вот как теперь выглядит линк `base_link`:

```xml
<link name="base_link">
  <!-- Визуальная часть (как и раньше) -->
  <visual>
    <geometry>
      <cylinder length="0.6" radius="0.2"/>
    </geometry>
    <material name="blue">
      <color rgba="0 0 .8 1"/>
    </material>
  </visual>

  <!-- НОВОЕ: Коллизионная часть -->
  <collision>
    <geometry>
      <cylinder length="0.6" radius="0.2"/>  <!-- Часто копирует visual -->
    </geometry>
    <!-- Можно задать смещение <origin .../>, если нужно -->
  </collision>
</link>
```

#### **1.1 Разбор кода коллизии**
*   Тег **`<collision>`** находится на том же уровне, что и `<visual>`, внутри `<link>`.
*   Его структура повторяет `<visual>`: есть теги `<geometry>` и опционально `<origin>`.
*   **Зачем копировать визуальную геометрию?** В простых случаях коллизионная форма совпадает с видимой. Но часто её **упрощают** по двум причинам:
    1.  **Скорость расчёта:** Проверять столкновения для двух сложных 3D-мешей гораздо тяжелее, чем для простых геометрических примитивов (цилиндров, коробок, сфер).
    2.  **Зоны безопасности:** Можно сделать коллизионную геометрию больше, чем реальная деталь, чтобы создать "безопасную зону". Например, обернуть голову робота цилиндром, чтобы ничего не могло к ней приблизиться вплотную.

### **2. Добавление инерционных свойств (`<inertial>`)**

Теперь добавим массу и инерцию:

```xml
<link name="base_link">
  <visual> ... </visual>
  <collision> ... </collision>

  <!-- НОВОЕ: Инерционные характеристики -->
  <inertial>
    <mass value="10"/>                   <!-- Масса в килограммах -->
    <inertia ixx="1e-3" ixy="0.0" ixz="0.0"
             iyy="1e-3" iyz="0.0"
             izz="1e-3" />               <!-- Тензор инерции -->
  </inertial>
</link>
```

#### **2.1 Масса (`<mass>`)**
*   Задаётся в **килограммах**. Значение должно быть реалистичным для вашей детали.

#### **2.2 Тензор инерции (`<inertia>`)**
Это самая сложная часть для новичков. Тензор инерции — это матрица 3x3, которая описывает, как масса распределена относительно осей координат. Для симметричных объектов её можно **рассчитать** или взять из CAD-программ.

**Как это читать:**
*   `ixx`, `iyy`, `izz` — моменты инерции относительно осей X, Y, Z. Для вытянутого цилиндра (как наше основание) момент относительно продольной оси (Z) будет меньше, чем относительно поперечных (X и Y). В примере все три равны `1e-3` — это **упрощение**.
*   `ixy`, `ixz`, `iyz` — центробежные моменты. Для симметричных объектов относительно осей координат они обычно равны нулю.

**Откуда брать значения?**
*   **Для простых форм (цилиндр, куб, сфера)** можно вычислить по формулам из Википедии (список моментов инерции).
*   **Для сложных мешей** — использовать программы вроде MeshLab, которые могут рассчитать инерцию по 3D-модели.
*   **Если совсем нет данных**, используйте разумные приближения. Для небольшой детали (размером ~0.1 м и массой ~0.6 кг) `ixx/iyy/izz = 1e-3` — хорошее начало. **Никогда не используйте единичную матрицу (1.0)!** Это соответствует нереально тяжёлому объекту (сотни килограмм).

> **Важно для контроллеров:** Если инерция будет близка к нулю, модель робота в симуляции может "схлопнуться" — все линки упадут в начало координат.

#### **2.3 Центр масс (`<origin>` внутри `<inertial>`)**
По умолчанию центр масс совпадает с началом координат линка. Вы можете изменить это, добавив тег `<origin>` внутрь `<inertial>`:

```xml
<inertial>
  <origin xyz="0 0 0.1" rpy="0 0 0"/> <!-- Центр масс смещён на 10 см вверх -->
  <mass value="10"/>
  <inertia .../>
</inertial>
```

### **3. Добавление динамики суставов (`<dynamics>`)**

Для подвижных суставов (`revolute`, `prismatic`, `continuous`) можно задать физические параметры движения с помощью тега `<dynamics>` внутри `<joint>`.

```xml
<joint name="head_swivel" type="continuous">
  <parent link="base_link"/>
  <child link="head"/>
  <axis xyz="0 0 1"/>
  <origin xyz="0 0 0.3"/>

  <!-- НОВОЕ: Динамические свойства сустава -->
  <dynamics friction="1.0" damping="0.5"/>
</joint>
```

*   **`friction`** — статическое трение. Для вращательных суставов (`revolute`, `continuous`) единицы измерения — **Н·м** (ньютон-метры). Для поступательных (`prismatic`) — **Н** (ньютоны).
*   **`damping`** — демпфирование (вязкое трение). Для вращательных — **Н·м·с/рад**, для поступательных — **Н·с/м**.

Если эти атрибуты не указаны, они по умолчанию равны **0** (нет трения и демпфирования).

---

## Ключевые выводы
- **Коллизионная геометрия (`<collision>`)** необходима для обнаружения столкновений в симуляторах. Её следует упрощать для производительности.
- **Инерционные свойства (`<inertial>`)** обязательны для физической симуляции. Включают массу и тензор инерции.
- **Тензор инерции** — сложная, но важная характеристика. Для простых форм его можно вычислить, для сложных — получить из CAD. Не используйте единичную матрицу.
- **Динамика суставов (`<dynamics>`)** позволяет задать трение и демпфирование, делая движение более реалистичным.
- Без этих свойств модель робота непригодна для симуляции в Gazebo и подобных средах.

## Практическое задание
1.  Возьмите любой линк из вашей модели (например, колесо или голову) и добавьте для него `<collision>` и `<inertial>`. Для простоты используйте те же геометрические примитивы, что и в `<visual>`.
2.  Рассчитайте тензор инерции для цилиндра, используя формулы:
    *   `ixx = iyy = (1/12) * mass * (3*radius^2 + height^2)`
    *   `izz = (1/2) * mass * radius^2`
    Подставьте массу 10 кг, радиус 0.2 м, высоту 0.6 м и сравните с упрощённым `1e-3`.
3.  Добавьте тег `<dynamics>` в один из суставов и поэкспериментируйте со значениями `friction` и `damping` в симуляторе (если есть возможность).
