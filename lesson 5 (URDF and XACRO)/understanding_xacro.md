# **Туториал по ROS 2: Используем Xacro для очистки URDF-кода**
*Учимся использовать константы, математику и макросы, чтобы сделать описание робота компактным и удобным*

## 🎯 **Цель туториала**
Познакомиться с языком макросов **Xacro** и научиться применять его три главные возможности для упрощения URDF-файлов:
1.  **Константы** — чтобы не повторять одни и те же числа.
2.  **Математика** — чтобы вычислять параметры на лету.
3.  **Макросы** — чтобы создавать переиспользуемые блоки кода.

---

## 📖 **Зачем нужен Xacro?**
Если вы писали URDF-файлы вручную, то наверняка сталкивались с проблемами:
*   **Повторение.** Одни и те же размеры (длина, радиус) приходится указывать много раз — в визуальной части, в коллизионной, в инерциальной.
*   **Сложность изменений.** Если нужно поменять длину ноги робота, приходится искать и править это число в десятке мест.
*   **Много кода.** Создание симметричных частей (левая и правая нога) требует копирования больших блоков с минимальными правками.

**Xacro** решает все эти проблемы. Это макроязык для XML, который позволяет определять константы, вычислять выражения и создавать макросы. Затем `xacro`-программа преобразует файл с макросами в чистый URDF, готовый к использованию.

## 🚀 **Практическая часть: Превращаем громоздкий URDF в элегантный Xacro**

### **1. Как использовать Xacro**
Самый простой способ — запустить `xacro` в командной строке, передав входной файл и получив выходной URDF:
```bash
xacro model.xacro > model.urdf
```

Но чаще всего это делают прямо в launch-файле, чтобы всегда использовать актуальную версию модели. Пример для `robot_state_publisher`:

```python
from launch.substitutions import Command
# ...
path_to_urdf = get_package_share_path('my_robot_description') / 'urdf' / 'robot.xacro'
robot_state_publisher_node = Node(
    package='robot_state_publisher',
    executable='robot_state_publisher',
    parameters=[{
        'robot_description': ParameterValue(
            Command(['xacro ', str(path_to_urdf)]), value_type=str
        )
    }]
)
```

А ещё проще — использовать готовый пакет `urdf_launch`, который сам найдёт и обработает xacro-файл.

В начале любого xacro-файла обязательно нужно объявить пространство имён:
```xml
<?xml version="1.0"?>
<robot xmlns:xacro="http://www.ros.org/wiki/xacro" name="my_robot">
```

### **2. Константы (Properties)**
Посмотрите на типичный фрагмент URDF для основания R2D2:
```xml
<link name="base_link">
  <visual>
    <geometry>
      <cylinder length="0.6" radius="0.2"/>
    </geometry>
    <material name="blue"/>
  </visual>
  <collision>
    <geometry>
      <cylinder length="0.6" radius="0.2"/>
    </geometry>
  </collision>
</link>
```
Дважды указаны одни и те же `length` и `radius`. Если мы решим изменить размер, придётся править оба места. С Xacro мы определяем константы один раз и используем их везде:

```xml
<xacro:property name="width" value="0.2" />
<xacro:property name="bodylen" value="0.6" />

<link name="base_link">
  <visual>
    <geometry>
      <cylinder radius="${width}" length="${bodylen}"/>
    </geometry>
    <material name="blue"/>
  </visual>
  <collision>
    <geometry>
      <cylinder radius="${width}" length="${bodylen}"/>
    </geometry>
  </collision>
</link>
```
Теперь изменение размера требует правки всего двух строк вверху файла.

**Синтаксис:**
*   Определение свойства: `<xacro:property name="ИМЯ" value="ЗНАЧЕНИЕ" />`
*   Использование: `${ИМЯ}`

Можно использовать константы и внутри текста, не только в числах:
```xml
<xacro:property name="robotname" value="marvin" />
<link name="${robotname}s_leg" />   <!-- Превратится в <link name="marvins_leg" /> -->
```

### **3. Математика (Math)**
Внутри `${...}` можно писать целые выражения с использованием `+`, `-`, `*`, `/`, унарного минуса и скобок.

```xml
<!-- Радиус колеса — половина диаметра -->
<cylinder radius="${wheeldiam/2}" length="0.1"/>

<!-- Координата с учётом отражения и зазора -->
<origin xyz="${reflect*(width + 0.02)} 0 0.25" />
```
Доступны и математические функции, такие как `sin`, `cos`, `pi`.

### **4. Макросы (Macros)**
Макросы — самая мощная часть Xacro. Они позволяют создавать свои собственные XML-теги, которые будут разворачиваться в целые блоки кода.

#### **4.1 Простой макрос**
```xml
<xacro:macro name="default_origin">
  <origin xyz="0 0 0" rpy="0 0 0"/>
</xacro:macro>

<!-- Использование: -->
<xacro:default_origin />
```
Этот код сгенерирует `<origin rpy="0 0 0" xyz="0 0 0"/>`. Бесполезно, но демонстрирует идею.

#### **4.2 Параметризованный макрос**
Гораздо полезнее макросы, которые принимают параметры. Например, стандартный инерциальный блок:

```xml
<xacro:macro name="default_inertial" params="mass">
  <inertial>
    <mass value="${mass}" />
    <inertia ixx="1e-3" ixy="0.0" ixz="0.0"
             iyy="1e-3" iyz="0.0"
             izz="1e-3" />
  </inertial>
</xacro:macro>

<!-- Использование: -->
<xacro:default_inertial mass="10"/>
```

#### **4.3 Макрос с блочными параметрами**
Иногда нужно передать в макрос не просто число, а целый XML-блок. Для этого имя параметра отмечают звёздочкой `*`, а вставляют блок с помощью `<xacro:insert_block>`.

```xml
<xacro:macro name="blue_shape" params="name *shape">
  <link name="${name}">
    <visual>
      <geometry>
        <xacro:insert_block name="shape" />
      </geometry>
      <material name="blue"/>
    </visual>
    <collision>
      <geometry>
        <xacro:insert_block name="shape" />
      </geometry>
    </collision>
  </link>
</xacro:macro>

<!-- Использование: -->
<xacro:blue_shape name="base_link">
  <cylinder radius=".42" length=".01" />
</xacro:blue_shape>
```
Этот макрос создаст линк с именем `base_link`, синим цветом, и вставит переданный блок (`<cylinder ...>`) и в визуальную, и в коллизионную части.

### **5. Практический пример: ноги R2D2**
Посмотрим, как с помощью макросов элегантно создаются две симметричные ноги.

```xml
<xacro:property name="leglen" value="0.5" />
<xacro:property name="width" value="0.2" />
<xacro:property name="pi" value="3.1415926535" />

<xacro:macro name="leg" params="prefix reflect">
  <!-- Сам линк ноги -->
  <link name="${prefix}_leg">
    <visual>
      <geometry>
        <box size="${leglen} 0.1 0.2"/>
      </geometry>
      <!-- Смещаем так, чтобы нога свисала вниз -->
      <origin xyz="0 0 -${leglen/2}" rpy="0 ${pi/2} 0"/>
      <material name="white"/>
    </visual>
    <collision>
      <geometry>
        <box size="${leglen} 0.1 0.2"/>
      </geometry>
      <origin xyz="0 0 -${leglen/2}" rpy="0 ${pi/2} 0"/>
    </collision>
    <xacro:default_inertial mass="10"/>
  </link>

  <!-- Сустав, крепящий ногу к основанию -->
  <joint name="base_to_${prefix}_leg" type="fixed">
    <parent link="base_link"/>
    <child link="${prefix}_leg"/>
    <!-- reflect = 1 для правой ноги, -1 для левой. Так они оказываются по разные стороны -->
    <origin xyz="0 ${reflect*(width + 0.02)} 0.25" />
  </joint>
</xacro:macro>

<!-- Создаём две ноги одним макросом -->
<xacro:leg prefix="right" reflect="1" />
<xacro:leg prefix="left" reflect="-1" />
```

**Что здесь происходит:**
*   **Параметр `prefix`** позволяет создавать линки и суставы с разными именами: `right_leg` и `left_leg`.
*   **Параметр `reflect`** принимает значение `1` или `-1`. В суставе мы умножаем его на смещение, чтобы поместить ноги симметрично относительно центра.
*   **Математика в `<origin ...>`** `-${leglen/2}` сдвигает ногу так, чтобы её верхний край был на уровне крепления, а сам линк свисал вниз.
*   **Макрос `default_inertial`** (определённый ранее) добавляет инерцию без лишнего копирования.

### **6. Запуск xacro-модели**
В пакете `urdf_tutorial` есть готовый пример. Чтобы увидеть результат работы макросов, выполните:

```bash
ros2 launch urdf_tutorial display.launch.py model:=urdf/08-macroed.urdf.xacro
```
Откроется `rviz` с уже знакомой моделью R2D2. В терминале, где запущен launch-файл, можно увидеть, что `xacro` выполняется автоматически.

## 📝 **Ключевые выводы**
✅ **Xacro** — это макроязык, который делает URDF-файлы компактными и лёгкими в поддержке.
✅ **Константы (`<xacro:property>`)** позволяют задать значение один раз и использовать его многократно.
✅ **Математические выражения** в `${}` позволяют вычислять параметры на основе констант.
✅ **Макросы (`<xacro:macro>`)** позволяют создавать параметризуемые шаблоны кода, избегая копирования.
✅ **Блочные параметры (`*param`)** позволяют передавать в макрос целые фрагменты XML.
✅ Симметричные части робота удобно создавать с помощью параметров-отражателей (`reflect`).
