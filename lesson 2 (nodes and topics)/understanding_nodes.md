# Понимание нод в ROS 2

## Что такое ноды в ROS 2?

### **1. Граф ROS 2**
В следующих туториалах вы познакомитесь с основными концепциями ROS 2, которые составляют **"Граф ROS 2"**.

**Граф ROS 2** — это сеть элементов ROS, которые обрабатывают данные одновременно. Он включает все исполняемые файлы и соединения между ними, если их отобразить визуально.

### **2. Ноды в ROS 2**
Каждая нода в ROS должна выполнять **единственную, модульную задачу**, например:
- Управление двигателями колёс
- Публикация данных с датчика расстояния
- Обработка изображений с камеры

![Ноды и их взаимодействие](images/Nodes-TopicandService.gif)
*Ноды обмениваются данными через топики и сервисы*

Каждая нода может:
- Отправлять данные другим нодам через **топики**
- Предоставлять **сервисы** (запрос-ответ)
- Использовать **действия (actions)** для длительных операций
- Работать с **параметрами**

**Полная роботизированная система** состоит из множества нод, работающих согласованно. В ROS 2 один исполняемый файл (программа на C++, Python и т.д.) может содержать **одну или несколько нод**.

## Практическая часть

### **1. Запуск нод с помощью `ros2 run`**
Команда `ros2 run` запускает исполняемый файл из пакета:

```bash
ros2 run <имя_пакета> <имя_исполняемого_файла>
```

**Запустим turtlesim:**
```bash
ros2 run turtlesim turtlesim_node
```
Откроется окно turtlesim (как в предыдущем туториале).

Здесь:
- **Имя пакета:** `turtlesim`
- **Имя исполняемого файла:** `turtlesim_node`

Но мы ещё не знаем **имя ноды**. Узнать его можно с помощью `ros2 node list`.

### **2. Список всех активных нод**
```bash
ros2 node list
```

Эта команда показывает имена всех запущенных нод. Особенно полезна, когда:
- Нужно взаимодействовать с конкретной нодой
- Система запускает много нод и нужно отслеживать их состояние

**Пример работы:**
1. Запустите turtlesim в первом терминале
2. В новом терминале выполните:
   ```bash
   ros2 node list
   ```
   Результат: `/turtlesim`

3. Запустите телеоперацию в третьем терминале:
   ```bash
   ros2 run turtlesim turtle_teleop_key
   ```

4. Снова выполните `ros2 node list`:
   ```bash
   /turtlesim
   /teleop_turtle
   ```

Теперь видно две активные ноды.

#### **2.1 Переназначение (Remapping)**
Remapping позволяет изменять свойства нод по умолчанию:
- Имя ноды
- Имена топиков
- Имена сервисов
- и другие параметры

**Пример переназначения имени ноды:**
```bash
ros2 run turtlesim turtlesim_node --ros-args --remap __node:=my_turtle
```

Теперь, если выполнить `ros2 node list`, увидим:
```bash
/my_turtle
/turtlesim
/teleop_turtle
```

Откроется второе окно turtlesim с новым именем ноды.

### **3. Полная информация о ноде**
Узнав имена нод, можно получить детальную информацию о каждой:

```bash
ros2 node info <имя_ноды>
```

**Пример для нашей новой ноды:**
```bash
ros2 node info /my_turtle
```

Вывод покажет:
```
/my_turtle
  Subscribers:
    /parameter_events: rcl_interfaces/msg/ParameterEvent
    /turtle1/cmd_vel: geometry_msgs/msg/Twist
  Publishers:
    /parameter_events: rcl_interfaces/msg/ParameterEvent
    /rosout: rcl_interfaces/msg/Log
    /turtle1/color_sensor: turtlesim/msg/Color
    /turtle1/pose: turtlesim/msg/Pose
  Service Servers:
    /clear: std_srvs/srv/Empty
    /kill: turtlesim/srv/Kill
    ...
  Action Servers:
    /turtle1/rotate_absolute: turtlesim/action/RotateAbsolute
  Action Clients:
```

**Что показывает `ros2 node info`:**
- **Subscribers (подписчики):** топики, на которые подписана нода
- **Publishers (издатели):** топики, в которые публикует нода
- **Service Servers:** сервисы, которые предоставляет нода
- **Service Clients:** сервисы, которые использует нода
- **Action Servers:** actions, которые предоставляет нода
- **Action Clients:** actions, которые использует нода

**Сравните информацию для разных нод:**
```bash
ros2 node info /teleop_turtle
ros2 node info /turtlesim
```

Обратите внимание, как различаются их подключения в графе ROS!

---

## Ключевые выводы
- **Нода** — фундаментальный элемент ROS 2, выполняющий одну модульную задачу.
- **Один исполняемый файл** может содержать несколько нод.
- **`ros2 node list`** показывает все активные ноды в системе.
- **`ros2 node info`** даёт полную информацию о подключениях ноды.
- **Remapping** позволяет переназначать свойства нод при запуске.
