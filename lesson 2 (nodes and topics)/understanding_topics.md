# Понимание топиков в ROS 2

## Что такое топики?
В ROS 2 сложные системы разбиваются на множество нод. **Топики** — это виртуальные каналы, по которым ноды обмениваются сообщениями.

**Нода может:**
- Публиковать данные в несколько топиков
- Подписываться на несколько топиков одновременно

**Топики позволяют** перемещать данные между разными частями системы.

## Предварительные требования
1. Знакомство с нодами ROS 2 (из предыдущего туториала)
2. Установленный ROS 2
3. Не забывайте выполнять `source` в каждом новом терминале!

---

## Практическая часть

### **1. Запускаем turtlesim**
```bash
# Терминал 1:
ros2 run turtlesim turtlesim_node

# Терминал 2:
ros2 run turtlesim turtle_teleop_key
```
Теперь работают две ноды: `/turtlesim` и `/teleop_turtle`.

### **2. Визуализация графа (rqt_graph)**
```bash
ros2 run rqt_graph rqt_graph
```
Или через интерфейс: **Plugins → Introspection → Node Graph**

![rqt_graph визуализация](images/rqt_graph.png)
*Граф нод и топиков в turtlesim*

**Что видим:**
- `/teleop_turtle` публикует команды в топик `/turtle1/cmd_vel`
- `/turtlesim` подписан на этот топик и получает команды
- **Совет:** наведите курсор на топик, чтобы увидеть соединения

### **3. Список всех топиков**
```bash
# Простой список
ros2 topic list

# Список с типами сообщений
ros2 topic list -t
```
Вы увидите примерно:
```
/turtle1/cmd_vel [geometry_msgs/msg/Twist]
/turtle1/color_sensor [turtlesim/msg/Color]
/turtle1/pose [turtlesim/msg/Pose]
```

### **4. Просмотр данных в топике**
```bash
ros2 topic echo /turtle1/cmd_vel
```
Теперь двигайте черепашку в терминале с `turtle_teleop_key` и наблюдайте данные в реальном времени!

![Отображение echo в rqt_graph](images/debug.png)
*rqt_graph с включенным echo*

### **5. Информация о топике**
```bash
# Базовая информация
ros2 topic info /turtle1/cmd_vel

# Подробная информация (показывает QoS и т.д.)
ros2 topic info /turtle1/cmd_vel --verbose
```

### **6. Структура сообщений**
Каждый топик имеет определённый тип сообщения. Узнаем структуру:
```bash
ros2 interface show geometry_msgs/msg/Twist
```
Вывод показывает, что сообщение содержит:
- `linear` (x, y, z)
- `angular` (x, y, z)

### **7. Публикация в топик вручную**
```bash
# Непрерывная публикация (1 раз в секунду)
ros2 topic pub /turtle1/cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 2.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 1.8}}"
```

![Непрерывная публикация](images/pub_stream.png)
*Непрерывная публикация команд*

```bash
# Однократная публикация
ros2 topic pub --once /turtle1/cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 2.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 1.8}}"
```

![Однократная публикация](images/pub_once.png)
*Однократная публикация и движение черепашки*

![rqt_graph после публикации](images/rqt_graph2.png)
*rqt_graph после ручной публикации*

### **8. Измерение частоты публикаций**
```bash
ros2 topic hz /turtle1/pose
```
Показывает, как часто публикуются данные в топик `/turtle1/pose`.

### **9. Измерение используемой пропускной способности**
```bash
ros2 topic bw /turtle1/pose
```
Показывает, сколько данных передаётся через топик.

### **10. Поиск топиков по типу**
```bash
ros2 topic find geometry_msgs/msg/Twist
```
Находит все топики, использующие указанный тип сообщения.

---

## 🧹 **Очистка**
Не забудьте остановить все процессы, нажав **Ctrl+C** в каждом терминале.

---

## Ключевые выводы
- **Топики** — основной способ обмена данными между нодами ROS 2.
- Каждый топик имеет **определённый тип сообщения**.
- **Один топик** может иметь **несколько издателей и подписчиков**.
- ROS 2 предоставляет **множество инструментов** для интроспекции топиков.

---

## Что дальше
Освоив топики, переходите к изучению **сервисов (services)** — другому типу коммуникации в ROS 2, который работает по принципу "запрос-ответ".

## Практическое задание
1. Запустите turtlesim
2. Создайте свою ноду-subscriber на топик `/turtle1/pose`
3. Измерьте частоту обновления позиции черепашки
4. Попробуйте опубликовать команду, чтобы черепашка двигалась по кругу

Следующий шаг — написать собственные ноды publisher и subscriber и проверить обмен сообщениями на практике.
